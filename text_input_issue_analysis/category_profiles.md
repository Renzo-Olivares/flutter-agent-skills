## How to read this file

Each category section reports a fixed set of stats. Definitions:

- **Size** — number of issues assigned to the category (counts match `text_input_taxonomy.json`).
- **Reactions** — sum and per-type breakdown of GitHub reactions across the category's issues.
- **Ownership** — `team-*` label distribution, issue-count descending; reactions shown as a secondary column. `orphaned` = no `team-*` label.
- **Priority** — counts of `P0`/`P1`/`P2`/`P3`; `unlabeled` = issue has no priority label.
- **Platform labels** — distribution of `platform-*` labels; these are orthogonal to `team-*` ownership, and only ~61% of issues carry one.
- **Issue-type tilt** — bucketed `c: *` labels. `bug` = `crash`/`fatal crash`/`regression`/`performance`/`flake`. `feature` = `proposal`/`new feature`/`new widget`. `other-c-label` = any other `c: *`. `no c: label` = issue has none.
- **Age** — days since `created_at`, measured against the script-run time. `>3y old` uses a 1095-day threshold.
- **Recency** — issues whose `updated_at` is within the last 90 days.
- **Reproducibility** — share of issues with the `has reproducible steps` label (corpus-wide: ~52%).
- **Found in release (top 3)** — most-cited `found in release: X.Y` labels.
- **Top 5 issues by reactions** — hyperlinked examples, reaction-total descending.
