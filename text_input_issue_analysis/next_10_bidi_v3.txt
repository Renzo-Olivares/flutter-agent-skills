  Comment: The issue behaves differently across platforms.

On Flutter Desktop (Windows/Linux/macOS), dynamically changing textDirection on an active TextField can crash the Flutter Engine. During the rebuild of the native text input layout, the engine attempts to convert a wide string (`std::wstring`) to UTF-8 using `wstring_convert::to_bytes`.
If the text is in an intermediate or invalid Unicode state (for example, during IME composition or layout recalculation), this conversion fails and throws a std::range_error with the message:
```
wstring_convert: to_bytes error
```


Since the Flutter Engine is built with -fno-exceptions, the exception cannot be caught, resulting in an application abort.

On mobile platforms (Android/iOS), the app usually does not crash because a different text input pipeline is used. However, changing textDirection during active text composition may invalidate the IME composing state, causing unsupported or incomplete Unicode sequences to be replaced with a fallback character (**displayed as ? or �**).
