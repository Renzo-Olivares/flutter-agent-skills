The confirmed workaround is to manually replicate the exact `TextStyle` the `TextField` applies (e.g., resolving from the material theme). @LongCatIsLooong mentioned a planned framework-level API exposing immutable text layout metrics via a callback (e.g., `onTextLayoutChanged`) but noted a `TextEditingController` cannot be the right place since one controller can serve multiple fields with different styles. Summary: Root cause identified as the default directional focus key bindings in `app.dart` being enabled on macOS (and potentially Linux).
